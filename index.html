The Origin of Web Systems

The story begins in 1989 with Tim Berners-Lee, a British scientist. He invented the World Wide Web to help scientists share information easily.

His first web system had three main technologies:

  • HTML: A simple language to create linked documents.
  • HTTP: A set of rules for computers to request and send these documents.
  • URL: A unique address for every document.

From this simple start, web systems have evolved into the complex, interactive platforms we use every day.

What is a Web System?

The Complete Team Behind a Website

A web system is the complete team of parts working together to bring a website to your screen. This includes computers (servers), code, networks, and databases.

What is its Role?

Its role is to power our digital world. Web systems run everything from e-commerce (Shopee) and social media (Facebook) to education and online banking.

Check for Understanding

Besides the website you see, can you name one other component that must be part of a web system to store your login information?

A database. It's a specialized component on the server-side responsible for securely storing and managing data like user accounts, passwords, and other information.

Web System vs. Website

Are they interchangeable?

In casual conversation, people often use "website" to mean the whole thing. In a technical context, "web system" is more accurate because it includes all the backend components, not just the frontend pages.

Check for Understanding

Is an e-commerce site like Shopee, where you add items to a cart, a static or a dynamic web application? Why?

It's a dynamic web application. Because it allows you to perform actions (add to cart, checkout), manages data specific to you (your cart), and the content changes based on your interaction, it must be generated dynamically.

Client-Side vs. Server-Side

Client-Side (Frontend)

This refers to everything the user sees and interacts with in their browser. It's the "front" of the house, responsible for the user interface and visual presentation.

Server-Side (Backend)

This is the "behind-the-scenes" part of the system that runs on a remote computer. It handles data, security, and the core logic of the application.

The Client-Server Model

Check for Understanding

When you see a loading spinner on a website after clicking a button, which side (client or server) is likely doing the most work at that moment?

The server-side. The client has sent a request and is now waiting for the server to process it (e.g., fetch data, perform a calculation) and send back a response. The loading spinner is a client-side indicator that this background process is happening.

Client-Side Technologies

HTML

HyperText Markup Language is the standard for creating web pages. It uses "tags" to define elements like headings, paragraphs, links, and images, forming the essential structure of the content.

CSS

Cascading Style Sheets is the language used to describe the presentation of a document written in HTML. It controls the layout, colors, fonts, and overall visual appearance.

JavaScript

A programming language that enables interactive web pages. It is used for animations, validating forms, and dynamically updating content without reloading the page.

Analogy: The Frontend as a Human Body

HTML is the Skeleton

Provides the core structure and framework of the body.

CSS is the Appearance

Controls the visual presentation, like our skin, hair, and clothes.

JavaScript is the Muscles

Enables movement and action, like our nervous system telling our muscles to move.

Check for Understanding

If you wanted to change the color of a button on a webpage, which core client-side technology would you use?

CSS. While HTML creates the button element itself, CSS is responsible for all visual styling, including its color, size, and position.

What is a Framework?

Think about baking a cake. You could source the flour, sugar, and eggs yourself (writing code from scratch), or you could use a cake mix (a framework).

A framework is a pre-built set of tools, code, and conventions that gives developers a head start. It provides a structure for their application so they don't have to build everything from zero, saving time and reducing errors.

Popular Frontend Frameworks

React

A popular library for building user interfaces. Developed by Facebook.

Angular

A comprehensive framework for building large-scale applications.

Vue.js

A progressive framework that is known for being easy to learn and flexible.

Server-Side Technologies

Programming Languages

Code (e.g., Python, PHP, Node.js) that runs on the server to process user input, interact with databases, and perform the application's core logic.

Databases

Organized collections of data (e.g., MySQL, MongoDB) that store all the necessary information, from user profiles to product listings.

Web Servers

Software (e.g., Apache, Nginx) that accepts HTTP requests from clients, fetches the requested resources, and sends them back to the client.

Analogy: The Backend as a Human Body

Languages: The Brain

Processes requests and makes decisions, like our conscious thought.

Databases: The Memory

Stores and retrieves all information, just like our long-term memory.

Web Servers: The Senses

Listens for and responds to requests, like our senses receiving information.

Check for Understanding

When you create a new account, which backend component is responsible for saving your username and password?

The Database. The backend programming language takes your information and sends it to the database to be stored securely for future logins.

Why Learn About Web Systems?

As an IT student, understanding web systems is a fundamental skill. It's the foundation for many careers in technology.

  • Career Opportunities: It opens doors to jobs like Frontend, Backend, or Full-Stack development.
  • Problem-Solving: It teaches you how complex systems work together to solve problems.
  • Digital Literacy: In a world that runs on the web, understanding its architecture makes you a more informed and capable digital citizen.

Summary of Key Concepts

  • A Web System is the complete ecosystem of hardware and software that delivers a web experience.
  • Web Applications are interactive, while Websites are mostly informational.
  • The Client-Side is what you see in your browser (Frontend); the Server-Side is the "brain" that runs remotely (Backend).
  • The core technologies are HTML (structure), CSS (style), and JavaScript (interactivity).
