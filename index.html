Lesson Development - Defining a Web System

Definition:

A web system is a comprehensive software architecture that enables the delivery of information and functionality over the Internet, typically accessed via a web browser. It is a collection of interconnected components working synergistically.

Core Purpose:

  • To serve dynamic content (text, images, video).
  • To facilitate user interaction (data input, transactions).
  • To manage and process data (user profiles, product catalogs, financial records).

Real-world Importance:

  • Communication: Social media, email, messaging.
  • Commerce: Online shopping, banking, financial services.
  • Education: E-learning platforms, online research.
  • Information Access: News portals, encyclopedias, search engines.
  • Entertainment: Streaming, online gaming.

Web systems are fundamental to the modern digital economy and societal infrastructure.

Websites vs. Web Applications: A Clear Distinction

Website:

  • Definition: Primarily a collection of static or mostly static web pages (documents) accessible via a uniform resource locator (URL). Its main purpose is to present information.
  • Characteristics: Less interactive, content-focused, often uses HTML and CSS for display.
  • Examples: Personal blogs, news sites, company information pages.

Web Application:

  • Definition: A dynamic, interactive application that is delivered over the internet via a web browser. It offers user interaction, data processing, and often integrates with databases.
  • Characteristics: Highly interactive, functionality-driven, processes user input, stores and retrieves data.
  • Examples: Online banking systems, e-commerce platforms (e.g., Amazon, Lazada), social media platforms (e.g., Facebook, Twitter), online productivity suites (e.g., Google Docs).

Key Difference: The degree of interactivity and dynamic functionality. Websites *inform*; web applications *perform* actions.

Client-Side vs. Server-Side Processing: The Two Halves

Client-Side Processing:

  • Location: Occurs directly within the user's web browser or device.
  • Function: Handles the presentation layer, user interface, and immediate user interactions.
  • Tasks: Rendering HTML, applying CSS styles, executing JavaScript for animations, form validation, managing user interface elements.
  • Benefit: Reduces server load, provides immediate feedback to the user.

Server-Side Processing:

  • Location: Occurs on a remote server that hosts the web system.
  • Function: Manages business logic, data storage and retrieval, security, and authentication. It responds to client requests by generating dynamic content or performing operations.
  • Tasks: Processing form submissions, accessing databases, user authentication, generating dynamic HTML pages, handling complex calculations, API integrations.
  • Benefit: Centralized data management, enhanced security, scalability.

Interaction: The client sends requests to the server, and the server processes them, sends back responses, which the client then renders.

Overview of Key Technologies in Web Systems

  • 1. HyperText Markup Language (HTML):
    • Role: The standard markup language for creating web pages. It provides the structure and content of a webpage.
    • Analogy: The skeleton of a house.
  • 2. Cascading Style Sheets (CSS):
    • Role: A style sheet language used for describing the presentation of a document written in HTML. It controls the visual appearance.
    • Analogy: The paint, wallpaper, and interior design of a house.
  • 3. JavaScript (JS):
    • Role: Enables interactivity on web pages. Allows for dynamic content, animations, form validation, and complex user interfaces.
    • Analogy: The electrical wiring and smart home features.
  • 4. Web Servers:
    • Role: Store web content and deliver it to web browsers upon request.
    • Examples: Apache HTTP Server, Nginx.
  • 5. Databases:
    • Role: Organized collections of data. Crucial for storing dynamic content, user information, etc.
    • Examples: MySQL, PostgreSQL, MongoDB.
  • 6. Server-Side Programming Languages:
    • Role: Executed on the web server to process requests, interact with databases, handle business logic.
    • Examples: Python, PHP, Node.js, Ruby, Java.
