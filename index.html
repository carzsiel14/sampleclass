The Origin of Web Systems

The story begins in 1989 with Tim Berners-Lee, a British scientist at CERN. He invented the World Wide Web (which is not the same as the internet!).

He proposed a system to share information across a computer network. This first web system had three main technologies:

  • HTML: A language to create web pages.
  • HTTP: A protocol to send and receive web pages.
  • URL: An address to find each web page.

From this simple start, web systems have evolved into the complex, interactive platforms we use every day.

What is a Web System?

A web system is a comprehensive framework of software, hardware, and network protocols that work in unison to deliver content and services over the internet. It encompasses everything from the physical servers storing the data to the code running in your browser.

Think of it as the complete ecosystem that allows a website or application to function, communicate, and serve its purpose to users worldwide.

Check for Understanding

Besides the website you see, can you name one other component that must be part of a web system to store your login information?

A database. It's a specialized component on the server-side responsible for securely storing and managing data like user accounts, passwords, and other information.

Website vs. Web Application

Website (Static)

A website is primarily informational. Its main goal is to present content that doesn't change based on user interaction. It's like a digital brochure or a book—you can read it, but you can't change its content.

📰

Examples: A company's informational page, a blog, a portfolio site.

Web Application (Dynamic)

A web application is interactive. It's designed for user engagement, allowing you to perform tasks, create content, and manipulate data. It's more like a software program that you access through your browser.

🛒

Examples: Online banking, social media (Facebook), e-commerce (Shopee).

Check for Understanding

Is an e-commerce site like Shopee, where you add items to a cart, a website or a web application? Why?

It's a web application. Because it allows you to perform actions (add to cart, checkout), manages data specific to you (your cart), and changes based on your interaction, it is a dynamic application.

Client-Side vs. Server-Side

Every web system has two fundamental sides that communicate constantly to deliver the experience you see on your screen.

Check for Understanding

When you see a loading spinner on a website after clicking a button, which side (client or server) is likely doing the most work at that moment?

The server-side. The client has sent a request and is now waiting for the server to process it (e.g., fetch data, perform a calculation) and send back a response. The loading spinner is a client-side indicator that this background process is happening.

The Request-Response Cycle

This is the fundamental conversation that happens every time you interact with a website.

  1. The Request: Your browser (the Client) sends a request over the internet to the server's address (e.g., www.google.com). This request asks for a specific resource, like a webpage, image, or data.
  2. The Processing: The Server receives the request. It finds the required resource, processes any necessary logic (like checking if you're logged in), and gathers data from a database if needed.
  3. The Response: The Server sends the resource back to your browser as a response, typically in the form of HTML, CSS, and JavaScript files.
  4. The Rendering: Your browser receives these files and renders them into the visual webpage you see and interact with.

Client-Side (Frontend)

This is everything the user sees and interacts with directly in the browser. Its primary job is to present data and capture user input.

HTML (HyperText Markup Language)

The core structure and skeleton of a webpage. It defines the elements like headings, paragraphs, images, and links.

CSS (Cascading Style Sheets)

Controls the visual presentation. It handles the layout, colors, fonts, and overall aesthetic of the website to make it visually appealing.

JavaScript

Brings the page to life. It adds interactivity, such as animations, form validations, pop-up messages, and dynamic content updates without needing to reload the page.

A Quick Look at Frontend Frameworks

For complex web applications, developers often use frameworks—pre-written code libraries—to build user interfaces more efficiently.

React

A popular library for building user interfaces with reusable components. Developed by Facebook.

Angular

A comprehensive framework for building large-scale applications. Developed by Google.

Vue.js

A progressive framework that is known for being easy to learn and flexible.

Check for Understanding

If you wanted to change the color of a button on a webpage, which of the three core client-side technologies would you use?

You would use CSS. While HTML creates the button element itself, CSS is responsible for all styling, including its color, size, and position.

Server-Side (Backend)

This is the engine room of the web system. It runs on a remote computer (the server) and handles all the complex logic, data management, and security.

Programming Languages

These languages (like Python, PHP, Node.js) process user requests, perform calculations, and execute the core business logic of the application.

Databases

Databases (like MySQL, MongoDB) are responsible for storing, organizing, and retrieving all the application's data, such as user profiles, products, or posts.

Web Servers

This is the software (like Apache or Nginx) that listens for incoming requests from clients (browsers) and serves them the appropriate files and data.

Check for Understanding

When you create a new account on a website, which backend component is responsible for saving your username and password?

The Database. The backend programming language takes your information and sends it to the database to be stored securely for future logins.

The Role of APIs

APIs (Application Programming Interfaces) are the messengers that allow different software components to talk to each other.

In a web system, the frontend (client) doesn't talk directly to the database. Instead, it sends a request to a backend API. The API then gets the data from the database and sends it back to the frontend in a structured format (like JSON).

This is crucial for security and organization. It also allows mobile apps and other services to use the same backend data as the website.

Putting It All Together

Let's trace a simple action: Liking a post on social media.

  1. Client: You click the "Like" button (HTML element).
  2. Client: JavaScript code runs, instantly changing the button's color to give you feedback. It then sends an API request to the server, saying "User 123 likes Post 456".
  3. Server: The web server (Nginx) receives the request and passes it to the backend application (e.g., Python code).
  4. Server: The backend code validates the request (Are you logged in? Does the post exist?) and then tells the database to update the "likes" count for Post 456.
  5. Server: The database confirms the update was successful. The server sends back a success message to your browser.
  6. Client: Your browser receives the success message. The "like" is now permanently saved.

Summary of Key Concepts

  • A Web System is the complete ecosystem of hardware and software that delivers a web experience.
  • Web Applications are interactive, while Websites are mostly informational.
  • The Client-Side is what you see in your browser (Frontend); the Server-Side is the "brain" that runs remotely (Backend).
  • The core technologies are HTML (structure), CSS (style), and JavaScript (interactivity).
  • The server uses programming languages, databases, and web server software to function.
  • APIs act as messengers between the client and the server.

Career Paths in Web Development

Understanding these concepts opens doors to many exciting careers.

Frontend Developer

Specializes in the client-side. They use HTML, CSS, and JavaScript to build the user interface and ensure a great user experience.

Backend Developer

Specializes in the server-side. They build and maintain the server, application logic, and databases, focusing on performance and security.

Full-Stack Developer

A "jack-of-all-trades" who is comfortable working on both the frontend and backend parts of a web system.
